Hi! Vertical Tabs Reloaded just got updated.

You can open the tab sidebar via hotkey (default: Ctrl+Shift+V) or the VTR icon next to the navigation bar.

Changelog Summary for 0.13.0 - Current release

  1. Minified pinned tabs Pinned tabs are now getting displayed as favicon-only (opt-out). This saves a lot of space for people with many pinned tabs.

  1. Improved visibility of favicons While using a dark theme, dark favicons are now getting inverted for better accessibility.

  1. Further improvements Tabs can now be dragged & dropped to the bookmark bar to create a new bookmark.
    Tab icons, like the sound indication and the close button, can now be displayed on the left instead of the right.

Changelog Summary for 0.10.0 - 0.12.1

  1. New Context Menu Choices The context menu now includes: Move tab to new window, reload all tabs, close all tabs below, close all tabs above, close all other tabs, bookmark all (visible) tabs, unload tab, duplicate tab and restore last closed tab.

  2. Better Interoperability Support of the tab hide API empowers you to use tab group add-ons or similar add-ons seamless with VTR. Also, VTR settings are getting synchronized via Firefox Sync across devices (opt-out).

  1. Extrem Performance Improvements Opening the sidebar with several thousand tabs could take several minutes with previous versions, now we are talking about several seconds. You can look at some benchmark results here.

  2. More Mouse Gestures Middle clicks on free space within the sidebar opens now a new tab. There is also a new option which enables to close tabs by middle clicking them (opt-in).

  1. Further improvements The tab context menu closes now automatically, when leaving the sidebar with the mouse and on scrolling. The close button can be displayed now on every tab. Tab tooltip can be configured. Many fixes regarding the private mode, multiple windows, tab closing, context menu and sidebar toggling. Also, it is possible again to change the hotkey which toggles the sidebar. Visual help for drag & drop got added.

Important changes in 0.9

With the release of Firefox Quantum 57 all add-ons were required to be re-written from scratch.

Due to technical changes, Vertical Tabs Reloaded may look and behave differently now. While many things did improve with this update, unfortunately, some functionality was required to be left behind, due to limitations from Firefox itself.
Luckily, most lost features will come back in the future.

Themes

All themes got updates.
The Dark and the Light themes got some fine turning, Darwin got a stronger identity, while the Linux and Windows themes got merged into a new theme named Feather. Feather is a bright, minimalistic theme.

Also finally: Pinned tabs are always visible.

More stable

Vertical Tabs Reloaded got a lot more stable overall.
No more crazy behavior. No more errors when opening tabs in fullscreen, disabling Vertical Tabs Reloaded temporary doesn't leave the tab browser broken. Clicks on tabs don't accidentally close them anymore. Overall, a lot of unwanted side effects are gone.

Lost features

Some features are not available anymore, due to technical limitations from Firefox:
* compact mode (tab icon only)
* tab sidebar can't be shown when in fullscreen
* horizontal tab browser can't be hidden
* no "native" theme
* tab toolbar can't be customized via Firefox customization page
* not compatible with many other add-ons like Tab Mix Plus

Roadmap

Many new features are on the road ahead, as well as re-adding removed features:

  • horizontal tab browser should be getting hidden
  • custom tab sidebar themes
  • a gallery for custom themes
  • new native theme
  • new compact tabs option
  • create an API to work nicely together with other add-ons (e. g. tab group add-ons and add-ons similar to Tab Mix Plus)
  • fixing bugs, all of these changes have for sure introduced new ones :)
  • and many, many other improvements...
